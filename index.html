About Segmentation

Segmentation divides memory into logical units called segments — Code, Stack, Heap, Data, etc. Each segment has a base address and a limit (size).


A Segmentation Fault occurs when a process tries to access memory outside its allocated segment, or when there isn't enough contiguous memory for the segment.

Physical Address = Base[s] + Offset
Valid if: Offset < Limit[s]
Else: SEGMENTATION FAULT

Theory & Viva Help

Key concepts, algorithm explanations, and exam preparation

Core Concepts

▦ VIRTUAL MEMORY

Virtual memory is a memory management technique that provides an idealized abstraction of the storage resources on a system. It gives each process the illusion of having a large, contiguous block of memory, even though physical RAM may be limited or fragmented. The OS maps virtual addresses to physical addresses using a page table.

Virtual Address Space >> Physical RAM
OS manages the mapping + swapping to/from disk

▦ PAGING

Paging divides logical memory into fixed-size blocks called pages, and physical memory into equally-sized frames. Pages are mapped to frames via a page table. When a process accesses a page not currently in a frame, a page fault occurs, and the OS loads the page from secondary storage (disk/swap).

Page Fault Rate = Faults / Total References
Hit Rate = 1 − Fault Rate

▦ DEMAND PAGING

In demand paging, pages are not pre-loaded. A page is loaded into memory only when it is first accessed (demanded). If the page is absent, a page fault is triggered. This avoids loading unnecessary pages and allows processes larger than physical RAM to run. Each page fault in this simulator represents a demand paging load event.

▦ FIFO (First In, First Out)

FIFO replaces the page that has been in memory the longest. A queue tracks insertion order. The oldest page is at the front and is evicted on a fault. Simple to implement but suffers from Bélády's Anomaly — more frames can sometimes cause more page faults.

Evict: oldest page in queue → replace with new page

▦ LRU (Least Recently Used)

LRU replaces the page that was least recently used. It uses recency of access as a proxy for future use. When a fault occurs, the page with the longest time since last access is evicted. LRU is a practical, widely-used algorithm that performs well in real workloads. It requires tracking usage history (timestamps or stack).

Evict: page with oldest last-use timestamp

▦ OPTIMAL (Bélády's Algorithm)

Optimal replaces the page that will not be used for the longest time in the future. It requires knowing the entire future reference string in advance, making it impractical for real systems. However, it serves as a theoretical benchmark — the minimum possible page faults for any algorithm given the same inputs.

Evict: page whose next use is farthest in the future (or never used again)

▦ SEGMENTATION

Segmentation divides a process's logical address space into variable-sized segments (Code, Data, Stack, Heap). Each segment has a base address and a limit. Segmentation reflects the programmer's view of memory and supports protection and sharing per segment. A segmentation fault occurs when access is outside a valid segment.

Physical Address = Segment Base + Offset
Condition: Offset < Segment Limit

▦ MEMORY FRAGMENTATION

Internal fragmentation occurs when allocated memory is larger than needed (fixed-size partitions waste the leftover space). External fragmentation occurs when there's enough total free memory for a request, but it's scattered in non-contiguous pieces. Compaction, paging, and segmentation can help mitigate external fragmentation.

Internal Frag = Allocated Block − Requested Size
External Frag = Scattered free blocks that can't be used
